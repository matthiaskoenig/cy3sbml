Short message    :	Arguments to <eq> and <neq> must have the same data types
Full message     :	[Although SBML Level 2 Version 1 does not explicitly define the following as an error, other Levels and/or Versions of SBML do.] 
The values of all arguments to <eq> and <neq> operators must have the same type (either all Boolean or all numeric). 
 The formula 'neq(gt(3, 6), c)' in the math element of the <event> with id 'e' uses arguments that should be either both numeric or both boolean.
